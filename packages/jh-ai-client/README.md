# @jh/ai-client

Minimal vanilla-JS client for invoking **JH AI Agent** from sibling JH applications
(JHEditor, JHER, JH Task Manager, …).

## Why this exists

The JH project ecosystem centralizes all AI logic in **JH AI Agent**. Every other JH
application is a "thin client" that:

1. Reads connection settings from a standard path (auto-written by JH AI Agent's
   "Export Connection" button).
2. Sends `{prompt, behavior, context}` requests.
3. Either waits for a single-shot response or streams an agent loop.

This package gives you all three for free. Drop it into any JS app — Tauri,
Electron, plain browser, Node — and you're connected.

## Install (workspace setup)

In each consuming JH app's `package.json`:

```json
{
  "dependencies": {
    "@jh/ai-client": "file:../jh-ai-agent/packages/jh-ai-client"
  }
}
```

Then `npm install`.

## Usage

### Setup (once per process)

```js
import JhAiClient from '@jh/ai-client';
import { invoke } from '@tauri-apps/api/core';
import { readTextFile } from '@tauri-apps/plugin-fs';

// In a Tauri context, supply readConfigFile so we can find the standard
// connection JSON. In a browser, pass {host, port, token} directly.
const ai = new JhAiClient({
  readConfigFile: async (path) => {
    // expand %APPDATA% / $HOME on the Tauri side, then read
    const expanded = path
      .replace('%APPDATA%', /* env */)
      .replace('$HOME', /* env */);
    return await readTextFile(expanded);
  },
});
await ai.ready();
```

### One-shot generation (JHER's SQL use case)

```js
const result = await ai.invoke({
  prompt: 'Find users with more than 5 orders last month',
  context: { schema: erSchemaJson },
  behavior: {
    mode: 'single_shot',
    system_prompt:
      'You are a PostgreSQL expert. Given a schema and a natural-language ' +
      'requirement, output one valid SQL query wrapped in a ```sql block. No prose.',
    response_format: 'code',
  },
  caller: 'JHER',
});

console.log(result.content);
// → "```sql\nSELECT u.* FROM users u JOIN orders o ON …\n```"
```

### Iterative agent (JHEditor's refactor use case)

```js
const task = ai.invokeAgent({
  prompt: 'Refactor the Store module to use TypeScript',
  workspacePath: '/path/to/repo',
  behavior: {
    mode: 'iterative_agent',
    // omit system_prompt → JH AI Agent uses its full built-in prompt with
    // anti-loop, verify_syntax, task_progress, etc.
    enabled_tools: null, // null = all tools (read_file, write_file, run_command, …)
  },
  caller: 'JHEditor',
  onStep: (pkt) => {
    if (pkt.event === 'thought') console.log('💭', pkt.data.text);
    if (pkt.event === 'tool_call') console.log('⚙', pkt.data.name);
  },
  onConfirm: async (req) => {
    // Show your own UI to ask the user; return true / false / modified string
    return await myApprovalUI(req);
  },
});

const result = await task.completed;
console.log('Done!', result.content);
console.log('Files modified:', result.modifiedFiles);

// To abort partway:
//   task.abort();
```

## Behavior reference

| Field                | Type            | Purpose                                              |
|----------------------|-----------------|------------------------------------------------------|
| `mode`               | string          | `"single_shot"` or `"iterative_agent"`               |
| `system_prompt`      | string          | Overrides built-in. Omit to inherit safety rules.    |
| `enabled_tools`      | string[] / null | Allowlist. null = all, [] = none, [...] = subset.    |
| `max_iterations`     | number          | Per-task cap. 0 = unlimited. Ignored in single_shot. |
| `response_format`    | string          | `"text"` / `"code"` / `"json"` (prompt-level hint)   |
| `extra_instructions` | string          | Appended to system_prompt (built-in or custom)       |

## Discovery

Connection details live in:

- **Windows**: `%APPDATA%/JH/ai-connection.json`
- **macOS**:   `$HOME/Library/Application Support/JH/ai-connection.json`
- **Linux**:   `$HOME/.config/JH/ai-connection.json`

Generated by JH AI Agent → Settings → General → **Export** button.
